Profile, Notifications & Settings

The signed-in customer's self-service account hub — edit personal details, read and clear the in-app inbox, and control language, notification opt-ins and password security.

ℹ️
Note — who sees these screens

These three screens live under the storefront Account menu, not the Admin console. Every one of them is strictly own-data only: the backend scopes every read and write to the verified signed-in user id, so a customer (or any staff member who is also a signed-in account holder) can only ever see and change their own record. A Super Admin previewing these pages sees their own profile — never another customer's. To edit a different customer's details as an operator, use Admin → Commerce → Customers instead.

Best Practice

Encourage customers to fill all three of name, email and phone. Beyond keeping invoices and delivery contact correct, a complete profile triggers a one-time DOODLY Pure Rewards bonus, and a verified email/phone is what makes password-reset and notifications actually reach them.

ℹ️
Note — read vs delete

Mark as read keeps the item but stops it counting as unread. Delete permanently removes it from the inbox. Both operations only ever touch rows that belong to the signed-in user, so there is no way to affect another customer's notifications.

Caution — both Security actions log you out everywhere

Updating the password and "Sign out of all devices" each bump the account's tokenVersion, which instantly invalidates every active session — including the one you are using. This is intentional (it locks out anyone else who was signed in), but it means you must sign in again straight afterwards. Warn customers before they click.

Best Practice

Set notification preferences with intent: keep at least one transactional channel (Email or WhatsApp) On so delivery and payment alerts still arrive, and use Promotions & offers as the single switch for marketing. If a customer complains about "too many messages," check this tab first — turning off Promotions usually solves it without silencing the essential delivery updates.

Frequently asked questions

Can a Super Admin edit another customer's profile from these screens?

No. Every request here is scoped to the signed-in user's own id, so these pages only ever show and change your own account. To edit another customer's details as an operator, use the Customers module in the Admin console.

Why does changing my password log me out of this device too?

Changing the password (and "Sign out of all devices") increments the account's token version, which invalidates every existing session for security. You'll be redirected to sign in again with the new password.

If I turn off WhatsApp or Email here, will I still get delivery alerts?

Those alerts respect your opt-ins, so turning a channel Off stops its messages. Keep at least one transactional channel (Email or WhatsApp) On so you still receive delivery and payment notifications; use the Promotions & offers toggle to silence marketing only.

What does completing my profile unlock?

Once your name, email and phone are all filled in and saved, DOODLY awards a one-time Pure Rewards "profile complete" loyalty bonus. It is idempotent, so it is granted only the first time all three are present.
