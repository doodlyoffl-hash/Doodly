Search Insights

See exactly what customers and staff type into the DOODLY global search — how often, what they find, what they don't, on which device — and curate the trending searches that greet every empty search box.

Every DOODLY page carries a single global search box (open it with Ctrl-K / ⌘K, the header magnifier, the dashboard search bar, or the mobile menu). Whenever a customer runs a search, gets no results, or clicks a result, that action is quietly logged to the database. Search Insights is the reporting cockpit for that stream: a live analytics dashboard, a searchable event ledger, and a manager for the "trending" chips shown when the box is empty. It reads from the SearchEvent and TrendingSearch tables through the /api/search/admin endpoint — no spreadsheets, no manual counting.

ℹ️
Note — two tabs, two permission levels

The screen has two tabs: Analytics (read-only reporting, open to any manager who can view Reports or the CMS) and Trending (editing the search chips, restricted to CMS editors — Super Admin, Admin and Marketing). The page opens on Analytics.

Tip — when the chart switches to months

Ranges of 45 days or less draw one bar per day; anything longer draws one bar per month for the last six months. So "Financial yr" and "All time" show a monthly shape, while the shorter presets show day-by-day detail.

Charts and keyword tables

Below the KPIs sit five visual panels. The Search volume trend is a bar chart of query counts over the range. Searches by device and Searches by scope are proportional bar breakdowns. Three tables — Top search keywords, Zero-result searches and Most clicked results — each list up to 12 rows ranked by frequency. Empty panels read "No data yet." rather than showing a blank box.

✅
Best Practice — mine the Zero-result table weekly

The Zero-result searches table is the single most valuable panel here: it is a free, honest list of demand you are not meeting. If "lassi" or "butter" keep appearing with no results, that is a signal to add a product, a synonym, or a help article. Turn recurring misses into action items every week.

Note — what "Scope" tells you

Scope records who was searching: public = a visitor who is not signed in, customer = a signed-in customer, admin = a staff member searching inside the admin panel. It is derived server-side from the session, so it cannot be spoofed by the browser.

⚠️
Warning — Hide vs. Delete

Only the customer-facing search overlay pulls active terms (the live storefront shows up to 8 active chips, ordered by sort order). Use Hide for a term you might bring back — a seasonal promo, for instance — so its history and position are kept. Use Delete only for terms you never want again.

⛔
Caution — Delete is irreversible

Delete removes the TrendingSearch row for good. There is no undo and no soft-delete; the only way back is to add the term again as a fresh entry (which lands at the end of the order). The confirm prompt is your last checkpoint.

How the data gets here

From a keystroke in the search box to a row on this dashboard

Purpose Understand what each metric is actually counting so you read the numbers correctly.
Access Super Admin — background reference Path n/a — automatic

Nobody enters this data by hand. The global search palette records three kinds of event automatically and posts them to /api/search/log, which writes a row to the SearchEvent table. Search Insights then reads and aggregates those rows.

ℹ️
Note — logging never blocks the customer

Analytics logging is fire-and-forget: if the logging call fails, the customer's search still works normally, and the browser also keeps a small offline cache (doodly-search in localStorage) so counts aren't lost while the backend is briefly unreachable. This means the numbers are highly reliable but not guaranteed to the last event.

Do these numbers include staff searches?

Yes. Staff searches inside the admin panel are logged with scope = admin. To see only customer behaviour, use the ledger's Scope filter and pick Customer (or Public for signed-out visitors).

Why is Success Rate all-time even when I pick "Last 7d"?

The volume-trend chart and the "range" figures follow the preset, but the headline KPIs like Total Searches, Success Rate and No-Result Searches are computed across all events for stability. Use the ledger with a date filter when you need a strictly range-bound view of individual events.

If I delete a trending term, does it remove those searches from analytics?

No. Trending terms and search events are separate. Deleting a chip only stops it being suggested; every past search anyone ran on that term stays in the analytics and the ledger.

Can I reorder trending chips?

New terms are added to the end of the list (next sort order), and the storefront shows up to 8 active terms in that order. There is no drag-to-reorder control on this screen; to move a term to the front, delete it and re-add it, or hide the ones you want to drop below it.

✅
Best Practice — align trending with what's live

Keep the trending chips in step with your current catalogue and promotions, and cross-check them against the Top search keywords table. Promoting a term customers already search for (or a new product you're launching) turns the empty search box into a zero-cost merchandising slot.
