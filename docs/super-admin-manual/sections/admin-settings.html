Settings

The single place to set DOODLY's brand identity, notification channels and login-security policy — saved to the DOODLY database and applied wherever the platform reads them.

ℹ️
Note

Settings is a general-purpose configuration store for the handful of platform values that have no dedicated screen of their own. It is organised into three groups — General, Notifications and Security. Everything else (delivery timings, pincodes, GST rates, referral rules, roles) lives on its own page and is only linked from here, never edited here.

General

Warning

Notifications

Four platform-wide switches that turn each messaging channel on or off. The panel carries a standing reminder: “Enable channels platform-wide. Actual sending still needs provider credentials (see remaining config).” — i.e. flipping a toggle on here does not by itself send anything; SMS/WhatsApp/Email/Push still require the provider keys to be configured.

Note

Every field defaults to a sensible value even before anyone saves, because the backend seeds defaults (brand DOODLY, email on, password length 8, session 30 min, 5 attempts). A field only stops using its default once you save an explicit value over it.

Best Practice

Treat the notification toggles as intent, not delivery. Turn a channel On here and confirm the provider credentials are configured before you rely on it — otherwise customers will silently receive nothing. When rolling out a new channel, enable it, send one test campaign from Admin → Content → Notifications, verify delivery, then announce it.

⛔
Caution

Lowering Password min length or raising Max login attempts weakens account security for every user. Only relax these deliberately, and prefer stricter values. Keep Legal company name and GSTIN exactly matching your registration — they feed straight into invoices, and a wrong value is reflected on tax documents.

FAQ

Why can Admins see Settings but not change anything?

By design. The permission matrix grants Admin settings:view only; the edit action (settings:edit) is reserved for Super Admin. Admins get full visibility for reference, but platform-level configuration is a single-owner responsibility.

I switched on WhatsApp but customers get no messages — why?

The toggles are master enable-flags only. Actual sending needs the provider (Twilio/Resend/etc.) credentials configured. Enable the channel here, then confirm the provider keys are set before relying on it.

Where do currency, GST rate and delivery times really come from?

Currency, time zone and date format live in Settings. The default GST rate and product tax live on GST Management; delivery cut-off/slots live on Delivery Settings. The bottom “Managed on their own pages” panel links straight to each of them.

Is every change tracked?

Yes. Each save writes an audit entry with action settings.update and the changed keys, viewable under Admin → System → Audit Logs.
