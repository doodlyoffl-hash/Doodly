Referral Management

The control room for the refer-a-friend programme — every code, every friendship it created, the one-time ₹100 rewards, and the fraud alerts that keep it honest. Approve, reject or reverse a reward; only the Super Admin (and finance roles) can touch the money.

ℹ️
Note — how a referral becomes a reward

A customer shares their code → a friend signs up with it → the friend buys a qualifying subscription (30-day or longer) → the relationship becomes Eligible → the ₹100 is credited to the referrer's wallet as Reward Credited. The reward is intentionally delayed until the qualifying purchase, which is what stops fraud.

⛔
Caution — Reverse moves real money

Reverse pulls the ₹100 back out of the customer's wallet. Only do it for a confirmed fraudulent or mistaken reward, and always type a reason — it is written to the audit log. If the customer has already spent the balance, coordinate with Finance before reversing.

⚠️
Permission

Approve, Reject and Reverse require manage rights (Super Admin / finance). Other roles that can open this screen see it read-only — the action buttons return 403 Forbidden.

Fraud Alerts

Automatic checks that block gamed rewards.

Purpose Review and clear the relationships the system flagged as suspicious before any money moves.

What the system checks automatically

CheckWhat it catches
Self-referralSomeone using their own code (same email/phone).
Duplicate accountThe same person creating a second account to farm rewards.
Circular referralTwo people referring each other in a loop.
Duplicate rewardAn attempt to credit the same relationship twice.
✅
Best Practice

Clear the Fraud Alerts tab (the red badge) before running a Bulk process. Bulk approval credits everything eligible — you want the flagged ones resolved first.

FAQ

Can one customer earn the reward many times?

Yes — once per successful friend. Each referral is a separate relationship; the reward is one-time per friend, not per referrer.

What happens to the reward if I reject after crediting?

Use Reverse (not Reject) once a reward is credited — Reverse is the action that pulls the ₹100 back from the wallet and marks the reward reversed. Both are audited.

Does the customer see any of this?

Customers only see their own side on Account → Referrals — their code, share link, friends referred and rewards earned. This admin screen is internal.
