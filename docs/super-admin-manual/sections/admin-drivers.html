Drivers

Onboard and manage your delivery executives — the people who carry the morning A2 buffalo milk from the dairy to each customer's door — with a live fleet dashboard, one-page profiles, zone and capacity settings, password resets and safe soft-deletes.

ℹ️
One person, several records — no duplicate data

A "driver" (the app calls them Delivery Executives) is not a single table. Each driver links a login User account (role DELIVERY_EXECUTIVE) to a Driver record, plus a Delivery Capacity (how many bottles they can carry), an Executive Status (are they available right now) and their Deliveries for the day. The Drivers screen stitches all of these together so you edit one profile and every part stays in step. Nothing here keeps a second copy of the delivery board.

Understanding the status badge

The Status column condenses two things — whether the account is switched on, and the driver's live availability — into one badge. The rules apply in this order:

BadgeColourShown when
InactiveGreyThe driver's Active flag is off (they are on file but not working)
SuspendedRedTheir login is locked (from the Suspended checkbox) — overrides availability
AvailableGreenActive and free — live status Available, or just returned to the dairy
BusyAmberOn a run — assigned, accepted or out for delivery
On leaveBlueThe driver has flagged a break in the driver app
OfflineGreyActive but not on shift / no live status reported
ℹ️
Availability comes from the driver, status from you

Available / Busy / On leave / Offline are reported by the driver's own app as they work. Inactive and Suspended are set by you from the Manage drawer. That is why a suspended driver always reads red no matter what their app says.

What "Add driver" creates

On save, DOODLY creates the login User (role DELIVERY_EXECUTIVE, marked to reset the password on first login) and the linked Driver record (Active, with the employee ID and vehicle number) together in one transaction — so you never end up with a half-made account. The action is written to the audit trail as driver.create.

⚠️
Warning — the temporary password is shown only in this dialog

Type a password you can pass to the driver securely, then note it before you close. They must change it the first time they log in. If it is lost, use Reset password in the Manage drawer to issue a fresh one.

ℹ️
Active vs Suspended — two different switches

Active is an operational flag: an inactive driver stops counting toward your fleet and won't be handed new stops, but their login still works. Suspended is a security lock: it blocks the person from signing in altogether (their badge goes red). Use Suspended when someone leaves or is under review; use Inactive for a temporary pause.

⚠️
Warning — Reset password shows the new password only once

After you click Reset password, the new temporary password appears in a toast message (for example "Temp password: Doodly@4821 — driver resets on next login"). Copy it and pass it to the driver right away — it is not stored in readable form and cannot be shown again. The driver is forced to set their own password the next time they log in.

ℹ️
Soft-delete, not permanent delete

Soft-delete hides the driver from the live list and marks the record deleted; the account and its delivery history are kept and can be restored. This screen never hard-deletes a driver, and deleting also switches them inactive. Restore is available through the backend (see below).

ℹ️
Bulk operations exist behind the scenes

The backend also supports acting on many drivers at once — activate, deactivate, assign a zone, soft-delete and restore (POST /api/admin/drivers/bulk; delete and restore need delete permission). The current Drivers screen manages one driver at a time; the bulk actions — including restore — are available to tooling built on the API.

✅
Best Practice

Set a realistic daily capacity for each driver (the default is 45 bottles). The auto-assignment engine caps how many stops a driver receives by this number, so an accurate figure keeps morning runs balanced and prevents anyone being overloaded or left idle.

Do I create the driver's login separately?

No. Add driver creates both the login account and the Driver profile together in one step. You only supply a temporary password, which the driver changes on their first login.

What is the difference between the Status badge and Availability?

Availability (Available / Busy / On leave / Offline) is reported by the driver's own app as they work. The Status badge combines that with the two switches you control — Inactive (Active off) and Suspended (login locked) — which always take priority. A suspended driver reads red regardless of their app.

Can I recover a driver I soft-deleted?

Yes. Soft-delete only hides and flags the record; the account and its history remain. Restoring is done through the backend's restore action (part of the bulk operations), which reactivates the driver and clears the deleted flag.

Why is a driver's capacity 45 when I never set it?

45 bottles is the default per-executive capacity. It applies until you set a specific number in the Manage drawer, at which point your value (1–10,000) is saved to that driver's Delivery Capacity.
