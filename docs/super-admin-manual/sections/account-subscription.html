My Subscription & History

The customer's self-service home for their recurring milk plan — see the live plan, pause or resume it, skip a day, switch auto-renew on or off, manage AutoPay, and read a timeline of every change and renewal.

ℹ️
Where this lives

These are customer-facing account pages served from the storefront (/account/subscription.html and /account/subscription-history.html). As a Super Admin you will use them when helping a customer over the phone, or after switching into a customer view. To manage anyone's subscription from the back office, use Admin → Commerce → Subscriptions (/admin/subscriptions.html) — a separate module. Every page here loads the signed-in customer's own data only; the panels stay blank (honest empty states) until a real account is signed in.

Two rendering paths

For a real signed-in customer the page fetches GET /api/account/subscription and draws a live "Your live plan" panel plus the real AutoPay mandate. For a guest / demo preview (not signed in) it instead shows a local sample "Active subscription" card so the layout can be explored. This manual documents the live behaviour, which is what a Super Admin sees when assisting a customer.

Buttons on the "Your live plan" panel

Which buttons appear depends on the current status. Every action calls POST /api/account/subscription with the subscription id and an action, then re-renders the panel with the fresh record.

ButtonWhat it doesBackend processPermission
Pause (only when Active)Asks "Pause for how many days?" (1–90). Deliveries resume automatically on the return date.action:"pause" with until = now + N days → status VACATION, sets pausedFrom = now and pausedUntil = untilCustomer

Resume deliveries (only when Paused / Vacation)Ends the pause and restarts deliveries.action:"resume" → status ACTIVE, clears pausedFrom and pausedUntilCustomer
Turn auto-renew on / offToggles automatic renewal. Label flips to match current state.action:"autopay_on" or "autopay_off" → sets autoRenew true / falseCustomer

⛔
Caution — Cancel plan is a one-way action

"Cancel plan" sets the subscription to CANCELLED and turns auto-renew off. There is no "un-cancel" — the customer must start a fresh subscription from /subscriptions.html. When in doubt, use Pause instead: it stops (and un-charges) deliveries but keeps the plan alive.

Every change is audited

The backend writes an audit record on each action — subscription.pause, subscription.resume, subscription.cancel, subscription.skip, subscription.autopay_on/off — tagged with the customer's user id and role. These feed the enterprise Audit Trail, so a Super Admin can always see who changed a plan and when.

When AutoPay isn't set up

If the subscription has no mandate the panel shows a friendly "AutoPay isn't set up for this subscription" state with a Start / renew a subscription button. AutoPay can only be enabled at checkout (the bank must authorise the e-mandate), so there is no "enable" button on this page.

Honest empty state

If the customer has never subscribed, the page keeps its seeded "No subscription history yet" message rather than inventing records. The client also has a fallback: if the dedicated history endpoint is unavailable it rebuilds a basic timeline from the plain subscription list, so the page never breaks.

✅
Best Practice

When a customer wants to stop deliveries for a trip, always steer them to Pause (with a resume date) rather than Cancel. Pause keeps their plan, pricing and AutoPay intact and auto-resumes on the chosen date; Cancel is permanent and forces a fresh signup. For a single day off, use Skip next delivery — it removes just that day and never charges for it.

Frequently asked questions

Q. What is the difference between "Cancel plan" and "Turn auto-renew off"?

A. "Turn auto-renew off" keeps the plan running for its remaining term but stops it renewing at the end. "Cancel plan" stops the subscription immediately (status CANCELLED, end date set today) and also turns auto-renew off — it cannot be undone.

Q. Will the customer be charged for a skipped or paused day?

A. No. Skipped days are added to the skipDates list and are never charged, and a paused (Vacation) plan makes no deliveries and no charges until it resumes.

Q. Can a Super Admin change a customer's plan from here?

A. Only by acting as that customer (their own account). To manage subscriptions across all customers from the back office, use Admin → Commerce → Subscriptions instead.
