Coupons

Create and manage discount codes — percentage or flat-value — that customers type at checkout to reduce their order total, with live usage, redemption limits and revenue tracking.

ℹ️
Where this lives

Open it from Admin → Growth → Coupons (URL /admin/coupons.html). The whole page is driven live from the DOODLY database through the endpoint /api/admin/coupons. When the page loads you will see a green banner reading “● Live — N coupon(s) from the DOODLY database”. If the banner is red, the backend is offline or your role is not allowed to view coupons — see Troubleshooting.

The coupon table

Every coupon is a row. The table is rendered by the shared DOODLY DataTable engine, so it supports search, sorting, filtering, export and pagination.

ElementDetail (from the real table config)
ColumnsCode (the code the customer types, e.g. FRESH10) · Description (the coupon's Name, plus its Campaign after a “·” if set) · Usage (times used, shown as used / limit when a total limit exists, otherwise just the count) · Status (colour badge) · Manage (opens the edit dialog).
Filter chipsAll · Active · Scheduled · Paused — quick facet filters above the table.
SearchFree-text box. The backend matches on code, name, campaign and description (case-insensitive).
SortingNewest first by default. The backend also supports sort by usage (most-redeemed), expiry (soonest), name (A→Z) and last-updated.
PaginationUp to 500 coupons are loaded into the page; the DataTable pages through them.
ExportThe toolbar's Export button downloads the visible table as CSV.
Row actionManage opens the Create/Edit Coupon dialog (next screen) pre-filled with that coupon.

A coupon's status is worked out automatically from its flags and dates (it is never typed in). The order of precedence is:

⚠️
“Paused” vs “Inactive”

The filter chip reads Paused, but a coupon you switch off shows the badge Inactive. They mean the same thing — a live coupon that has been turned off with the Active checkbox. Filtering by “Paused” finds your switched-off codes.

💡
Tip — flat vs percentage

Pick Flat ₹ when you want a fixed rupee cut (e.g. ₹50 off). Pick Percentage % for a proportional discount, and set Max discount so a big cart does not give away too much. A discount can never exceed the order total, so a flat ₹100 on a ₹80 cart only takes off ₹80.

⛔
Caution — deleting a live code

Delete is a soft-delete, so the redemption history is kept and you can Restore later — but the moment you delete, the code stops working at checkout. If a campaign is still running, switch Active off instead, or set an Expiry date, rather than deleting.

How a coupon is checked at checkout

When a customer types a code, the storefront calls the validation service, which runs these checks in order and stops at the first failure — giving the customer a clear reason. This is why a coupon that looks Active can still be refused for a specific cart.

#CheckMessage the customer sees when it fails
1Not deleted“This coupon no longer exists.”
2Active switch is on“This coupon is not active.”
3Start date reached“This coupon is not yet active.”
4Not past expiry“This coupon has expired.”
5Order meets minimum value“Your order does not meet the minimum value for this coupon.”
6First-order rule (if set)“This coupon is valid on your first order only.”
7Customer eligibility (if restricted)“This coupon is not available for your account.”
8Total usage limit not reached“This coupon has reached its usage limit.”
9Per-customer limit not reached“You have already used this coupon the maximum number of times.”
10Product / plan applies to the cart“This coupon does not apply to the items in your cart.”
11Discount is greater than zero“This coupon gives no discount on this order.”
ℹ️
Redemption is recorded once, at payment

Validation happens as the customer types the code, but the usage count only goes up when the order is actually paid. The redemption is recorded atomically and is idempotent per order — so a retried payment never double-counts a coupon.

✅
Best Practice

Always set a Usage limit and an Expiry date on promotional codes, and add a Per-customer limit of 1 for welcome or first-order offers. This caps your maximum discount exposure and stops a single code from being shared and over-used. For a percentage coupon, also set a Max discount ceiling so a large basket never gives away more than you intend. Use the Campaign field consistently so the reports can total each promotion cleanly.

Frequently asked questions

Q. What is the difference between a Coupon and an Offer?

A Coupon is a code the customer must type at checkout (managed here). An Offer is a campaign-driven promotion managed on the separate Admin → Growth → Offers screen. Use Coupons for shareable discount codes.

Q. Can I change a coupon after customers have started using it?

Yes. Editing and saving updates the rules for all future redemptions; past redemptions are untouched and stay in the coupon's history and totals.

Q. How do I run the same coupon for a new campaign without retyping everything?

Open the coupon and click Duplicate. You get an identical copy with a “-COPY” code, created switched off. Rename its code, adjust the dates, then tick Active.

Q. What happens to redemption history if I delete a coupon?

Nothing is lost. Delete is a soft-delete — the code stops working but its redemptions, discount given and revenue stay recorded, and you can Restore the coupon later.

Q. Why does the Usage column sometimes show “∞”?

When you leave the total Usage limit blank, the coupon can be redeemed an unlimited number of times, so the list shows only the count with no ceiling.
