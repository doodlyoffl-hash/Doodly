Attendance

Mark, correct and review the daily attendance of every DOODLY employee — one status per person per day, with instant present/absent counts, bulk marking, and a per-employee monthly calendar that feeds payroll.

ℹ️
Where this sits

Attendance is one screen in the Human Resources group (Admin → Human Resources → Attendance). It reads the Employee Master (only active staff appear), and what you record here flows straight into the monthly Payroll (payable days and overtime) and is written back automatically when a Leave request is approved. Every record is one row keyed to a single employee and a single date, so re-marking the same day simply overwrites the previous status — it never creates duplicates.

ℹ️
Marking twice = a correction, not a duplicate

Attendance is unique per employee per day. If a day already has a status and you choose a new one, the system overwrites it and records the change in the Audit Log as attendance.correct (showing the old and new status, e.g. "DL-014 2026-07-12 → ABSENT (was PRESENT)"). A first-time mark is logged as attendance.mark. Nothing is ever silently lost.

ℹ️
The month follows the register date

The calendar opens on the month of whatever date is selected in the register (for example, choosing 12 July shows all of July). To view a different month, change the register's date filter first, then reopen the calendar. The calendar is read-only — to change a day, close it and mark that day on the register.

ℹ️
The rule under the hood

Reading uses the view action; marking, correcting and bulk marking all use the edit action. A role granted only view (like Accountant) is blocked from every write, both in the interface and at the API. These defaults can be changed by a Super Admin in Roles & Permissions; the server matrix — not the browser — is the real security boundary.

✅
Best practice — mark daily, reconcile before payroll

Mark attendance every day (or use Bulk mark for the common case, then correct the exceptions), because Payroll counts payable days straight from these records: Present, WFH, Paid Leave and Sick Leave count in full, Half Day counts as 0.5, and overtime carries through in hours. Before running month-end payroll, open each employee's Monthly Calendar and clear any Unmarked days — an unmarked day is neither present nor absent and can distort the payslip. Remember that approving a Leave request automatically writes the matching status onto those days, so you usually only need to mark days leave doesn't cover.

Frequently asked questions

Can I mark attendance for a past date?

Yes. Set the register's Date to any past day and mark as usual — the record is keyed to that date. Re-marking a day that already has a status is logged as a correction, so the change is fully traceable in the Audit Log.

What is the difference between "Present" and "WFH"?

Both count as a present, payable working day and both show green. WFH simply records that the person worked from home rather than on site; it is a separate status so reports can distinguish the two.

How does Half Day affect pay?

Half Day counts as 0.5 of a payable day in the payroll calculation, whereas Present, WFH, Paid Leave and Sick Leave each count as a full payable day and Absent counts as zero.

Do I need to mark attendance for approved leave?

No. When a Leave request is fully approved, the system automatically stamps the correct status (Paid Leave, Sick Leave, or Loss-of-Pay as Absent) on every day of the leave. You only mark days that leave does not cover.

Is there an attendance report across a date range?

Yes, at the data level: the attendance service can produce a per-employee summary (present, half-day, absent, paid/sick leave, weekly off, holiday, overtime and payable days) over any From–To range, and this is exactly what Payroll consumes for the month. In this screen you export the single-day register as CSV or review a full month per employee via the Calendar.
